## 🐝 Project: HiveOps Dashboard Updates

Here is a concise summary of the architectural and logical improvements we made to the **Astro + Python** ETL pipeline for your apiary in Wołomin.

---

### 🏛️ Data Architecture
* **Split Storage:** Separated data into two distinct JSON files to optimize **Astro** build performance and payload size.
    * `telemetry.json`: High-resolution (3-hourly) data for the live dashboard.
    * `archive.json`: Daily summarized data for seasonal tracking and long-term GDD accumulation.
* **Stateful Rollover:** The script now detects local midnight by comparing the last entry's date with the current date, triggering a daily summary append to the archive.

---

### ⏱️ Timezone & Logic Precision
* **DST-Proof Time:** Replaced hardcoded UTC offsets with `ZoneInfo("Europe/Warsaw")`. This ensures the **Midnight Rollover** and **Date** strings are always aligned with the apiary's local clock, regardless of Daylight Savings.
* **Rolling 24h GDD:** Implemented a **Sliding Window** calculation. Every 3 hours, the script looks back exactly 24 hours to find $T_{max}$ and $T_{min}$, providing a smooth "heat momentum" line for the UI.
* **Unit Conversion:** Added a constant ($3.6$) to convert IMGW wind speed from **m/s** to **km/h** for better readability.

---

### 📉 Visualization Strategy
| Metric | UI Element | Purpose |
| :--- | :--- | :--- |
| **Current Temp** | Radial Gauge | Immediate hive status (Safe/Cold/Heat Stress). |
| **Rolling GDD** | Line Sparkline | Short-term heat momentum (24-hour window). |
| **Cumulative GDD** | Area Chart | Seasonal bloom predictor (Linden/Acacia countdown). |
| **Flight Status** | Status Badge | "Optimal" vs "Restricted" based on wind, rain, and temp. |

---

### 💻 Code Quality (Python)
* **SRP (Single Responsibility Principle):** Decoupled the `fetch` (I/O) from the `process` (Logic) layers.
* **Robustness:** Added defensive JSON loading to prevent script crashes if a file is corrupted or empty.
* **Metadata:** Included `timestamp` (ISO with offset) and `hour` fields to allow **Astro** components to render local times without complex client-side JS.



import requests
import json
import os
from datetime import datetime, timedelta
from zoneinfo import ZoneInfo

# --- Configuration ---
STATION_NAME = "warszawa"
BASE_TEMP = 10.0
TELEMETRY_FILE = "src/data/telemetry.json"
ARCHIVE_FILE = "src/data/archive.json"
MAX_TELEMETRY = 40  # ~5 days of 3-hourly data
POLAND_TZ = ZoneInfo("Europe/Warsaw")

def fetch_imgw_data():
    """Pure I/O: Fetches raw data from the IMGW API."""
    url = f"https://danepubliczne.imgw.pl/api/data/synop/station/{STATION_NAME}"
    try:
        r = requests.get(url, timeout=15)
        r.raise_for_status()
        return r.json()
    except Exception as e:
        print(f"Fetch Error: {e}")
        return None

def update_stores():
    """Processing Layer: Handles business logic, rolling windows, and archiving."""
    raw_data = fetch_imgw_data()
    if not raw_data:
        return

    # 1. Standardize Inputs & Logic
    now_pl = datetime.now(POLAND_TZ)
    t_now = float(raw_data['temperatura'])
    # Conversion: 1 m/s * 3.6 = 1 km/h
    wind_kmh = round(float(raw_data['predkosc_wiatru']) * 3.6, 1)
    rain_mm = float(raw_data['suma_opadu'] or 0)
    
    # Business Logic: Is it a good time to open the hive?
    is_flyable = t_now > 14 and wind_kmh < 20 and rain_mm == 0

    new_entry = {
        "timestamp": now_pl.isoformat(),
        "date": now_pl.strftime("%Y-%m-%d"),
        "temp": t_now,
        "wind": wind_kmh,
        "rain": rain_mm,
        "status": "Optimal" if is_flyable else "Restricted",
        "humidity": raw_data['wilgotnosc_wzgledna'],
        "pressure": raw_data['cisnienie']
    }

    # 2. Load Existing Data
    def load_json(path):
        if os.path.exists(path):
            with open(path, 'r') as f:
                try:
                    return json.load(f)
                except:
                    return []
        return []

    telemetry = load_json(TELEMETRY_FILE)
    archive = load_json(ARCHIVE_FILE)

    # 3. Daily Archive Rollover (Local Midnight)
    # If the stored date is older than today, finalize yesterday's stats
    if telemetry and telemetry[-1]['date'] != new_entry['date']:
        yesterday_date = telemetry[-1]['date']
        yesterday_data = [i for i in telemetry if i['date'] == yesterday_date]
        
        if yesterday_data:
            t_max = max(i['temp'] for i in yesterday_data)
            t_min = min(i['temp'] for i in yesterday_data)
            daily_gdd = max(((t_max + t_min) / 2) - BASE_TEMP, 0)
            
            last_cumulative = archive[-1]['cumulative_gdd'] if archive else 0
            
            archive.append({
                "date": yesterday_date,
                "t_max": t_max,
                "t_min": t_min,
                "daily_gdd": round(daily_gdd, 2),
                "cumulative_gdd": round(last_cumulative + daily_gdd, 2)
            })
            
            with open(ARCHIVE_FILE, 'w') as f:
                json.dump(archive, f, indent=2)

    # 4. Telemetry Update & Rolling 24h Window
    telemetry.append(new_entry)
    
    now_dt = datetime.fromisoformat(new_entry['timestamp'])
    window_start = now_dt - timedelta(hours=24)
    
    # Calculate Rolling GDD using the sliding 24-hour window
    window = [i for i in telemetry if datetime.fromisoformat(i['timestamp']) >= window_start]
    
    if window:
        r_max = max(i['temp'] for i in window)
        r_min = min(i['temp'] for i in window)
        telemetry[-1]['rolling_gdd'] = round(max(((r_max + r_min) / 2) - BASE_TEMP, 0), 2)
    else:
        telemetry[-1]['rolling_gdd'] = 0

    # 5. Maintain Buffer & Save
    if len(telemetry) > MAX_TELEMETRY:
        telemetry = telemetry[-MAX_TELEMETRY:]
        
    with open(TELEMETRY_FILE, 'w') as f:
        json.dump(telemetry, f, indent=2)

    print(f"Successfully updated {new_entry['timestamp']}. Status: {new_entry['status']}")

if __name__ == "__main__":
    update_stores()
